Tac:
A generic and adaptable interactive theorem prover

Overview

Tac is a framework for implementing interactive theorem provers with good support for both interactive proof development and automated proof search. While our main research focus is on automated reasoning, it is useful to be able to interactively play with the logic when developping automated tactics; moreover, an interactive system is often useful to manually fill the gaps between automatically discovered proofs. Currently, the main development within Tac is the logic μLJ and its automated focused inductive theorem proving tactic. Tac provides several tools:

  • Taci is a command line interface to the Tac core.
  • StickyTaci is a graphical user interface to Taci, written in C#. It allows end-users to easily interact with the Tac-based theorem provers, and provides such functionality as loading and saving of proofs, syntax highlighting, and automatic discovery of loaded logics and available tactics and tacticals.
  • Stekki is simpler graphic user interface, which should build on most UNIX-based systems; it is written in OCaml and uses Labltk.
  • Tac produces proofs in an XML format. The tool tactex can be used to convert them to LaTeX, and allows some basic focusing-based proof outlining.

More about Tac/μLJ

The logic μLJ is a simple but powerful extension of first-order logic with equality, least and greatest fixed points and minimal generic quantification. It enjoys a focusing proof system on which our implementation relies, for both automatic and interactive proof developments.

Learn more about μLJ in Focused Inductive Theorem Proving by Baelde, Miller, Snow & Viel (submitted paper).

The distributed system comes with several example theorems. They include proofs of properties of Peano arithmetic, standard list predicates, the pi-calculus, Buchi automata, and the simply typed lambda calculus.

Here is an illustrative list of the kinds of theorems for which the automated tactic prove can quickly find proofs:

  • Commutativity and associativity of addition on natural numbers.
  • Every natural number is either even or odd, the sum of two even numbers is even, double is the inverse of half for even numbers.
  • Reflexivity, symmetry and transitivity of inductively defined equalities on lists and natural numbers.
  • Reflexivity and transitivity of the ordering on natural numbers.
  • Left and right neutrality of nil for list concatenation.
  • Relationships between list reversal and length, list concatenation and length.
  • Antisymmetry of the subtree ordering on binary trees.
  • Reflexivity and transitivity of simulation.
  • Totality of the Ackermann function: pi x\ pi y\ nat x => nat y => sigma z\ ack x y z, nat z.
  • Totality of division by two: pi x\ nat x => sigma h\ half x h.
  • Involution of list reversal: pi x\ pi r\ reverse x r => reverse r x.

As said above, Tac produces proofs. Here are a couple examples. Most proofs are outlined: the asynchronous phase (invertible rules) is collapsed into one rule, leaving only the more relevant information in the proof. Even using this technique, we can only show here some of the simplest proofs automatically discovered. Larger proofs would be not only be unreadable, but in fact are not even manageable for LaTeX. Also, several of the Postcript files below are known to not work in Ghostview.

  • From arithmetic.muLJ.tac:
    • Commutativity of addition (PS).
    • Inductive equality on natural numbers if symmetric (PS).
    • The ≤ relationship on natural numbers is transitive (PS).
    • Truncated division by two (half) is total, with a hand-crafted invariant and only one induction (PS) or proved automatically using nested inductions (PS).
    • Every natural number is either even or odd, PS, see also the full proof (not outlined).
    • The Ackermann function is total (PS).
  • From sim.muLJ.tac: simulation is reflexive (PS) and transitive (PS).
  • From copy.muLJ.tac: inductive equality on λ-terms implies term equality, PS, PNG (large files, use a powerful viewer).
  • From lambda/lemmas.muLJ.tac: keys and variables found in a context that does not use a generic variable cannot use it either (PS).

Current developers
